# 🔭 おでかけスカウト（Odekake Scout）

**「今週末のおでかけ調べて」—— あとは全部、任せてください。**

子連れ家族のための週末おでかけ自動プランナー。近隣〜近県のイベントを自律的に検索し、家族構成・子どもの興味に合わせておすすめを選定、交通手段・ランチ・雨天対策まで含むおでかけプランを一括生成。さらに来月の先行予約情報まで調べ上げます。

▶️ [デモ動画（3分）](https://youtu.be/lktilaa-mEE)

## こんな経験、ありませんか？

- 「あのイベント、先週末だったの？知ってたら行ったのに…」
- 「ポケパークの予約、気づいたときには埋まってた…」
- 「子ども2人とも楽しめる場所って、毎回探すのが大変…」

おでかけスカウトは、こうした「知らなくて行けなかった」「調べる時間がなかった」をまるごと解決します。

## できること

Claude Code に「今週末のおでかけ調べて」と一言伝えるだけ。あとは追加指示なしで最後まで自走します。

1. **イベント調査**（scout） — 設定したエリアから子ども向けイベントをWeb検索で収集。結果が少なければクエリを自動で変えてリトライ
2. **おすすめ選定**（rating） — 天気予報を取得し、7つの評価軸でスコアリング。予約の取りやすさもWeb検索で確認した上で、おすすめ3件を自動選出
3. **おでかけプラン作成**（planner） — おすすめ3件それぞれについて、交通手段の自動判断（車/電車）・ルート・駐車場・子連れ向きランチスポット・雨天時の代替案を調査
4. **先行予約調査**（advance-scout） — 1〜2ヶ月先の大型イベントの予約開始日・締切・申込方法を調査し、緊急度別に整理
5. **レポート出力**（output） — すべてを統合したウィークリーレポートをMarkdownファイルとして保存

これらの5フェーズを、司令塔スキル（orchestrator）が途切れることなく一括実行します。約7〜10分で、プラン付きレポートと先行予約情報が手元に届きます。

## インストール

Claude Code で以下を実行してください。

```
/plugin marketplace add htanijiri/odekake-scout
/plugin install odekake-scout@odekake-scout-marketplace
```

## 初回セットアップ

インストール後、Claude Codeの**作業ディレクトリ**に `profile.yml` を作成してください。このファイルにご家族の情報を記載します。

```yaml
home_area: お住まいの地域（例: 葛飾区）
children:
  - name: 子どもの呼び名（1人目）
    age_group: 小学生
    interests: [好きなもの1, 好きなもの2]
  - name: 子どもの呼び名（2人目）
    age_group: 保育園児
    interests: [好きなもの]
search_areas: [検索したいエリア1, エリア2, エリア3]
max_travel_time: 片道1時間
transport:
  primary: 車
  secondary: 電車
  prefer_secondary_when:
    - 条件1
    - 条件2
```

### 記載例

#### 車メイン・電車サブの場合

```yaml
home_area: 葛飾区
children:
  - name: 長男
    age_group: 小学生
    interests: [マインクラフト]
  - name: 次男
    age_group: 保育園児
    interests: [ポケモン, 危険生物, 未確認生物]
search_areas: [東京23区, 千葉県, 埼玉県]
max_travel_time: 片道1時間
transport:
  primary: 車
  secondary: 電車
  prefer_secondary_when:
    - 都内の近場
    - 駐車場が不安な場所
    - 荷物が少ない時
```

#### 電車メイン・車サブの場合

```yaml
home_area: 世田谷区
children:
  - name: 娘
    age_group: 小学生
    interests: [サンリオ, お菓子作り]
search_areas: [東京23区, 神奈川県]
max_travel_time: 片道1時間
transport:
  primary: 電車
  secondary: 車
  prefer_secondary_when:
    - 郊外で電車のアクセスが悪い場所
    - 荷物が多い時
    - 雨の日
```

#### 電車のみ（車なし）の場合

```yaml
home_area: 中央区
children:
  - name: 息子
    age_group: 保育園児
    interests: [電車, 恐竜]
search_areas: [東京23区]
max_travel_time: 片道45分
transport:
  primary: 電車
```

`secondary` がない場合は、常に `primary` の交通手段でプランを作成します。

`profile.yml` が見つからない場合は、作成を促すメッセージが表示されます。

## 使い方

### 今週末のおでかけを調べる

```
今週末のおでかけを調べて
```

6つの Skill が連鎖して自動実行され、ウィークリーレポートが生成されます。

### 先行予約情報だけを調べる

```
来月のイベント予約を調べて
```

advance-scout が単独で起動し、1〜2ヶ月先の予約情報をまとめます。

### 訪問履歴の記録

レポート出力後に「海の森まつりに行ったよ」と伝えると、訪問履歴に記録されます。次回以降のおすすめに反映され、同じ場所が繰り返し推薦されるのを防ぎます。

## スコアリングの仕組み

7つの軸でイベントを評価します（満点: 55点）。

| 軸 | 重み | 内容 |
|---|---|---|
| 興味マッチ | ×3 | 子どもの登録済み興味との関連度 |
| 年齢適合 | ×2 | 兄弟全員が楽しめるか |
| 希少性 | ×2 | 今週末限りか、常設か。最終週や開催初週には補正あり |
| アクセス | ×1 | 自宅からの移動時間 |
| コスト | ×1 | 家族での総費用 |
| 天候リスク | ×1 | 屋内/屋外と天気予報の組み合わせ |
| 手軽さ | ×1 | 予約の要否・予約の取りやすさ |

加えて、予約困難なイベントは手軽さを減点、最近訪問した場所は合計スコアから減点します。これにより「実際に行ける、かつ今週末に行く価値があるイベント」が上位に来ます。

## 工夫したところ

- **「今週末しか行けないもの」を逃さない** — 希少性スコアにより、常設施設や長期開催より単発イベントが優先されます。ただし長期開催でも最終週なら加点されます
- **予約できないイベントはおすすめしない** — 予約状況をWeb検索で確認し、チケットが取れないイベントはスコアが下がります。おすすめされたのに行けない、というがっかりを防ぎます
- **車と電車を自動で使い分ける** — プロフィールの設定に応じて、メインの交通手段とサブの交通手段を条件に合わせて自動で切り替えます
- **来月の予約を先回り** — 今週末のプランだけでなく、1〜2ヶ月先の予約開始情報もまとめて出力。「気づいたら締切を過ぎてた」を防ぎます
- **行った場所を覚える** — 訪問履歴を記録することで、毎週同じ場所がおすすめされる問題を回避します
- **6つのSkillによる自律実行** — orchestrator（司令塔）が5つのフェーズを途切れずに一括実行。途中でユーザーの入力を必要としません

## アーキテクチャ

6つの Skill で構成されています。orchestrator が司令塔として全体を制御し、5つのフェーズを途切れずに一括実行します。

```
ユーザー: 「今週末のおでかけを調べて」
  ↓
orchestrator  司令塔（5フェーズを一括実行）
  ↓
scout         イベント調査（Web検索で収集・リトライ）
  ↓
rating        おすすめ選定（天気取得・7軸スコア・予約確認・訪問履歴）
  ↓
planner       プラン作成（交通手段・ランチ・雨天代替）
  ↓
advance-scout 先行予約調査（1〜2ヶ月先の予約情報）
  ↓
output        レポート出力（統合レポートをファイル保存）
```

advance-scout は単独でも起動可能です（「来月のイベント予約を調べて」）。

## 開発の背景

葛飾区在住、小学生と保育園児の2児の父が、自分の家族のために作りました。

毎週末「どこに連れて行こう」と検索するのが地味に大変で、しかも「あのイベント、先週だったんだ…」と後から知ることも多く、それなら毎週自動で調べてくれるエージェントを作ろうと思ったのがきっかけです。

profile.yml を書き換えれば、お住まいの地域やお子さんの興味に合わせてどなたでも使えます。
