# scout Skill — 今週末のイベント自動調査

## 役割

家族プロフィール（profile.yml）を読み込み、今週末の子ども向けイベントを Web 検索で自律的に収集する。
ユーザーは「今週末のおでかけ調べて」と言うだけでよい。聞き取りは行わず、プロフィール情報を元に自走する。

## 家族プロフィール

以下のコマンドを実行してprofile.ymlを読み込むこと。これは必須ステップである。

```
cat profile.yml
```

このコマンドでファイルの内容が表示されたら、その内容を家族プロフィールとして使用する。

ファイルが見つからない場合は、以下のメッセージを表示して処理を停止する:
「profile.yml が見つかりません。カレントディレクトリに profile.yml を作成してください。書き方の例は README.md の「カスタマイズ」セクションを参照してください。」

デフォルト値は使用しない。必ず profile.yml から読み込むこと。

## 検索戦略

1. **エリア別検索** — search_areas の各エリアで「{エリア} 子ども イベント 今週末」を Web 検索する
2. **興味別検索** — 子どもの interests を使い「{興味} イベント 関東」等で Web 検索する
3. **リトライ** — 結果が5件未満なら、クエリの言い回しを変えて最低3回は再検索する（例:「イベント」→「体験」→「ワークショップ」→「フェス」）
4. **構造化** — 各イベントの日時・場所・対象年齢・料金・申込要否・URL を抽出して整理する

## 出力

イベント一覧を以下の Markdown 形式で出力する。

```markdown
## 今週末のおでかけ候補

### 1. {イベント名}
- **日時**: 2026/3/21（土） 10:00〜16:00
- **場所**: {会場名}（{最寄駅 or エリア}）
- **対象年齢**: {対象}
- **料金**: {料金}
- **申込**: {要/不要/先着順}
- **URL**: {公式ページURL}
- **概要**: {1〜2行の説明}
```

## 完了後

【重要】イベント一覧の出力が完了したら、メッセージの表示やユーザーへの確認は一切行わず、即座にrating Skillの手順（天気予報の取得とスコアリング）を開始せよ。「続けて〜」「rating Skillを実行します」等のメッセージは絶対に表示しない。

## ルール

- **実在するイベントのみ**を掲載する。架空のイベント名や日程を絶対に作らない
- 検索結果をそのまま信頼せず、**日付が今週末かどうかを必ず確認**する。過去のイベントは除外する
- 結果が少ない場合は**最低3回はクエリを変えてリトライ**する
- 料金や申込情報が不明な場合は「要確認」と明記する
- 各イベントの公式ページ URL を必ず含める
- 最新情報は変更の可能性がある旨を末尾に注記する
