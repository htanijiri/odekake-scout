# planner Skill — おでかけプラン作成

おでかけスカウトの4つのスキル連鎖の3番目。
scout（イベント調査）→ rating（選定）→ **planner（計画作成）** → output（出力）

## 役割

- rating が選出した TOP3 のイベントそれぞれについて、具体的なおでかけプランを作成する
- 交通手段、ルート、ランチスポット、雨天時代替案をすべて Web 検索で調査する

## 家族プロフィール

profile.yml を探して読み込む。以下のコマンドで検索すること:

```
cat profile.yml 2>/dev/null || cat ${CLAUDE_PLUGIN_ROOT}/data/profile.yml 2>/dev/null
```

上記コマンドで内容が表示されたら、その内容をプロフィールとして使用する。

どちらも見つからない場合のみ、scout と同じデフォルト値を使用する。

## 交通手段の判断ロジック

profile.yml の transport 設定に基づいて自動判断する。

デフォルト:
```yaml
primary: 車
prefer_train_when:
  - 都内の近場（目的地が23区内で葛飾区から片道30分以内）
  - 駐車場が不安な場所（都心の繁華街、駐車場情報が見つからない場所）
  - 荷物が少ない時（屋内イベントで持ち物が少ない場合）
```

判断手順:
1. 目的地が23区内かつ片道30分以内 → 電車を検討
2. 目的地周辺の駐車場を Web 検索 → 見つからない or 高額 → 電車
3. それ以外 → 車
4. 判断理由を明記する

## 各プランに含める内容

1. タイムスケジュール（出発〜帰宅）
2. 交通手段と判断理由
3. ルート（車の場合は主要道路、電車の場合は路線と乗換）
4. 所要時間
5. ランチスポット（イベント会場周辺で Web 検索、子連れ向きの店を優先）
6. 雨天時の代替案（屋外イベントの場合、近くの屋内施設を Web 検索）
7. 持ち物メモ（イベント内容に応じて）

## 完了後

【重要】全プランの出力が完了したら、メッセージの表示やユーザーへの確認は一切行わず、即座にadvance-scout Skillの手順（1〜2ヶ月先のイベント予約情報の調査）を開始せよ。「続けて〜」等のメッセージは絶対に表示しない。

## ルール

- ランチスポットは必ず Web 検索して実在する店を提案する。架空の店名を作らない
- タイムスケジュールは子連れのペース（移動に余裕を持つ、昼食は長めに取る）を考慮する
- 駐車場情報は可能な限り調べて記載する（料金、台数）
- 雨天時代替案は屋外イベントの場合のみ記載する。屋内イベントなら「屋内のため雨天でも問題なし」と記載
- 各プランは独立して読めるようにする（他のプランを参照しない）
