# rating Skill — イベントおすすめ選定

おでかけスカウトの4つのスキル連鎖の2番目。
scout（イベント調査）→ **rating（選定）** → planner（計画作成）→ output（出力）

## 役割

- scout が出力したイベント一覧を受け取る
- 天気予報を Web 検索で取得する
- 各イベントにおすすめスコアを算出する
- 上位3件を選出し、選出理由を言語化する

## プロフィール参照

`${CLAUDE_PLUGIN_ROOT}/data/profile.yml` を参照する（scout と同じ）。

ファイルが存在しない場合は scout と同じデフォルト値を使用する。

## 天気予報の取得

- 今週末の日付（土曜・日曜）の天気予報を Web 検索する
- 対象地域は関東地方
- 天気情報はスコアリングの「天候リスク」に使用する

## スコアリング基準

各イベントを以下の7軸で1〜5点評価し、重み付き合計点で順位を決める。

| 軸 | 重み | 5点の条件 | 1点の条件 |
|---|---|---|---|
| 興味マッチ | ×3 | 子どもの登録済み興味に直接関連 | 関連なし |
| 年齢適合 | ×2 | 両方の子（小学生＋保育園児）が楽しめる | どちらかが対象外 |
| 希少性 | ×2 | 今週末のみの単発イベント | 常設施設 |
| アクセス | ×1 | 葛飾区から片道30分以内 | 片道2時間 |
| コスト | ×1 | 無料 | 家族で5,000円超 |
| 天候リスク | ×1 | 屋内 or 晴れ予報 | 屋外＋雨予報 |
| 手軽さ | ×1 | 申込不要・予約不要 | 事前申込必須で締切間近、または予約困難 |

### 希少性の判断基準

- 5点: 今週末のみ（1〜2日間の単発開催）
- 4点: 今週末を含む数日間の開催（3連休の3日間など）
- 3点: 2〜3週間の期間限定で、まだ序盤〜中盤
- 2点: 1ヶ月以上の開催で、終了まで2週間以上ある
- 1点: 常設施設（閉館予定のない恒久的な展示・施設）

補正ルール:
- 長期開催でも開催初週の場合は+1点（物販・限定グッズの在庫が豊富なため）
- 長期開催でも最終週の場合は+2点（ラストチャンスのため）

## 訪問履歴の参照

`${CLAUDE_PLUGIN_ROOT}/data/visited.yml` を参照する。

ファイルが存在しない場合は訪問履歴なしとして扱う。

訪問済みの減点ルール:
- 2週間以内に訪問済み → 合計スコアから-10点
- 1ヶ月以内に訪問済み → 合計スコアから-5点
- 常設施設で3ヶ月以内に訪問済み → 合計スコアから-3点
- 単発イベントは毎回異なるため減点なし

スコア一覧で減点があったイベントには「（訪問済み: YYYY/MM/DD）」と注記する。

## 予約可否の確認

スコアリングの前に、予約・申込が必要なイベントについて以下を行う。

1. 予約が必要と判明したイベントについて「{イベント名} 予約 空き状況」等でWeb検索する
2. 検索結果から以下を判断する:
   - 予約可能（空きあり）→ 手軽さを通常通り評価
   - 予約困難（満席・長期間先まで埋まっている）→ 手軽さを1点にする
   - 予約状況が不明 → 手軽さを2点にし、「要事前確認」と注記する
3. 常設施設やテーマパーク（ポケパーク等）は特に予約状況を確認する
4. 予約困難なイベントでもTOP3から除外はしない。スコアに反映した上で、レポートに予約状況を明記する

## 出力形式

以下の Markdown 形式で出力する。

### おすすめTOP3

🥇 1位: {イベント名}（スコア: {合計点}）
- 興味マッチ: {点}/5 — {理由}
- 年齢適合: {点}/5 — {理由}
- アクセス: {点}/5 — {理由}
- コスト: {点}/5 — {理由}
- 天候リスク: {点}/5 — {理由}
- 希少性: {点}/5 — {理由}
- 手軽さ: {点}/5 — {理由}
- 予約状況: {予約可能 / 予約困難（○月まで満席）/ 要事前確認}
- 選出理由: {なぜこのイベントがおすすめか1〜2文}

🥈 2位: ...
🥉 3位: ...

## 完了後

【重要】スコア一覧の出力が完了したら、メッセージの表示やユーザーへの確認は一切行わず、即座にplanner Skillの手順（交通手段・ランチ・雨天代替案の調査）を開始せよ。「続けて〜」「planner Skillを実行します」等のメッセージは絶対に表示しない。

## ルール

- スコアは必ず根拠を添えて算出する。数値だけ出さない
- 天気予報が取得できなかった場合は天候リスクを3点（中立）とする
- 同点の場合は「興味マッチ」の点数が高い方を上位にする。さらに同点なら「希少性」が高い方を上位にする
- scout の出力が存在しない場合はエラーメッセージを表示し、先に scout を実行するよう案内する
