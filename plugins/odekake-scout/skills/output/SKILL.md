# output Skill — ウィークリーレポート出力

おでかけスカウトの5つのスキル連鎖の最後。
scout（イベント調査）→ rating（選定）→ planner（計画作成）→ advance-scout（先行予約調査）→ **output（出力）**

## 役割

- scout、rating、planner の全出力を統合する
- 見やすいウィークリーレポートとして整形する
- Markdown ファイルとして保存する

## 出力形式

以下の構成でレポートを作成する。

```markdown
# おでかけスカウト ウィークリーレポート

**対象日: {YYYY/MM/DD}（土）〜{YYYY/MM/DD}（日）**
**作成日: {YYYY/MM/DD}**

---

## 🌤️ 週末の天気

- 土曜: {天気} / {最高気温}℃
- 日曜: {天気} / {最高気温}℃

---

## 🏆 おすすめTOP3 プラン付き

### 🥇 {イベント名}

おすすめスコア: {点}
選出理由: {理由}

{plannerが作成したプランをここにそのまま掲載}

---

### 🥈 {イベント名}

{同様}

---

### 🥉 {イベント名}

{同様}

---

## 📋 その他のイベント一覧

| # | イベント名 | 日時 | 場所 | スコア |
|---|---|---|---|---|
| 4 | ... | ... | ... | ... |

---

## 🔮 来月の先行予約情報

### 🔴 予約締切が近いもの
{advance-scoutの🔴セクションをそのまま掲載}

### 🟡 予約受付中
{advance-scoutの🟡セクションをそのまま掲載}

### 🟢 予約開始前
{advance-scoutの🟢セクションをそのまま掲載}

### 📋 予約不要の注目イベント
{advance-scoutの📋セクションをそのまま掲載}

---

## ℹ️ 注意事項

- イベント情報は検索時点のものです。お出かけ前に公式サイトで最新情報をご確認ください。
- 料金・申込状況は変更される場合があります。

---
*このレポートは おでかけスカウト（Odekake Scout）が自動生成しました。*
```

## 保存

- レポートを表示した後、ユーザーの作業ディレクトリに `{YYYY-MM-DD}-weekly-report.md` として保存を提案する

## 完了後

以下のメッセージを表示する:

> おでかけスカウト完了！ 上記のレポートをもとに、今週末のおでかけをお楽しみください。
> 気になるイベントの詳細や、別のイベントのプラン作成もお手伝いできます。

また、TOP3のイベントに行った場合は以下のように伝えてください:

> 行ったイベントがあれば教えてください。訪問履歴に記録して、次回以降のおすすめに反映します。

ユーザーが行ったイベント名を伝えたら、`${CLAUDE_PLUGIN_ROOT}/data/visited.yml` に以下の形式で追記する:

```yaml
visits:
  - name: "{イベント名}"
    date: "{訪問日 YYYY-MM-DD}"
    event_type: "{単発 / 長期開催 / 常設}"
```

## ルール

- TOP3 のプランは planner の出力をそのまま使い、内容を改変しない
- その他のイベント一覧はスコア順に並べる
- レポート全体が1つの Markdown ファイルとして成立するよう、見出しレベルを統一する
- 絵文字は見出しにのみ使用し、本文中では控える
- 予約困難なイベントがTOP3に含まれる場合は、注意事項セクションでその旨を目立つように記載する
- advance-scoutの出力は内容を改変せず、セクション構造をそのまま統合レポートに含める
- 先行予約情報がない場合は「来月の先行予約情報」セクション自体を省略する
