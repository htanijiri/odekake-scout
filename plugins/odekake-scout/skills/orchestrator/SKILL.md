# orchestrator — おでかけスカウト司令塔

ユーザーが「今週末のおでかけを調べて」と言ったら、以下の5つのスキルを Skill ツールで順番に呼び出す。

## 事前チェック

最初に profile.yml の存在を確認する。

```
cat profile.yml
```

ファイルが見つからない場合は、以下のメッセージを表示して処理を停止する:
「profile.yml が見つかりません。カレントディレクトリに profile.yml を作成してください。書き方の例は README.md の「カスタマイズ」セクションを参照してください。」

profile.yml が読み込めたら、以下のステップを順に実行する。

## 実行ステップ

### ステップ1: イベント調査

Skill ツールで `odekake-scout:scout` を実行する。

scout が今週末のイベント一覧を出力する。

### ステップ2: おすすめ選定

Skill ツールで `odekake-scout:rating` を実行する。

rating がイベントをスコアリングし、TOP3を選出する。

### ステップ3: プラン作成

Skill ツールで `odekake-scout:planner` を実行する。

planner がTOP3のイベントそれぞれについて具体的なおでかけプランを作成する。

### ステップ4: 先行予約調査

Skill ツールで `odekake-scout:advance-scout` を実行する。

advance-scout が1〜2ヶ月先の予約情報を調査する。

### ステップ5: レポート出力

Skill ツールで `odekake-scout:output` を実行する。

output が全フェーズの結果を統合してウィークリーレポートを生成・保存する。

## ルール

- 各ステップの完了後、次のスキルを呼び出す前にユーザーの入力を待たない
- 各スキルが出力した内容は、後続のスキルが会話コンテキストから参照する
- 5つのスキルすべてを実行し終わるまで処理を継続する
- 実在するイベントのみ掲載する。架空のイベントを作らない
- 検索結果の日付が今週末かどうかを必ず確認する
- ランチスポットも実在する店のみ提案する
- 料金や申込情報が不明な場合は「要確認」と明記する
