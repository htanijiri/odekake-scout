# advance-scout Skill — 先行予約調査

## 役割

1〜2ヶ月先に開催される子ども向けイベントのうち、事前予約や抽選申込が必要なものを調査し、予約情報をまとめて出力する。

## 家族プロフィール

profile.yml を探して読み込む。以下のコマンドで検索すること:

```
cat profile.yml 2>/dev/null || cat ${CLAUDE_PLUGIN_ROOT}/data/profile.yml 2>/dev/null
```

上記コマンドで内容が表示されたら、その内容をプロフィールとして使用する。

どちらも見つからない場合のみ、scout と同じデフォルト値を使用する。

## 検索戦略

1. 「{エリア} 子ども イベント {来月の月名}」で各エリアを検索
2. 「{興味} イベント {来月の月名} 関東 チケット 予約」で興味別に検索
3. 検索結果から予約・抽選が必要なイベントをピックアップ
4. 各イベントの予約開始日・申込方法・締切を調査
5. 結果が少なければクエリを変えてリトライ

## 出力形式

# 先行予約カレンダー（{来月の月名}のイベント）

## 🔴 予約締切が近いもの（1週間以内）

### {イベント名}

- 開催日: {日時}
- 場所: {場所}
- 予約締切: {日付} ← あと{X}日！
- 申込方法: {Web / 電話 / 抽選}
- 申込URL: {URL}
- 料金: {料金}
- 対象年齢: {対象}

## 🟡 予約受付中（締切まで余裕あり）

### {イベント名}

...

## 🟢 予約開始前（もうすぐ受付開始）

### {イベント名}

- 予約開始日: {日付}

...

## 📋 予約不要で行けるイベント

（予約不要の大型イベントも参考としてリスト化）

## ルール

- 実在するイベントのみ掲載する。架空のイベントや日程を作らない
- 予約開始日・締切が不明な場合は「要確認」と明記する
- 抽選制のイベントは当選確率の目安があれば記載する
- 公式ページのURLを必ず含める
- 検索結果をそのまま信頼せず、日付や料金は公式情報と照合する

## 完了後

【重要】先行予約情報の出力が完了したら、メッセージの表示やユーザーへの確認は一切行わず、即座にoutput Skillの手順（最終レポートの生成）を開始せよ。「続けて〜」等のメッセージは絶対に表示しない。
